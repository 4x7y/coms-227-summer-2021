Installing Java and Eclipse

In this course we will be using the Java Development Kit (JDK) along with a development called Eclipse for editing and debugging Java code. You will need Java version 11 (or higher). Note that Eclipse itself is a Java program, so you'll want to install Java first, then Eclipse.

Installing Java

If some version of Java is already installed on your system, it is likely just the "Java runtime environment", or JRE. This is enough to run Java programs, but for developing code we generally want the full Java Development Kit, or JDK (which also includes a JRE). There are many options for installing a JDK, but probably the simplest is the following:
  1. Go to https://adoptopenjdk.net/releases.html
  2. At the top, select checkboxes for OpenJDK 11 with the HotSpot JVM (later versions are probably fine too)
    • Exception: If you have a Mac and are running OS X Big Sur, you must use JDK version 14.
  3. Scroll down to the installer for your platform:
    • For Windows, scroll down to the x64 section and download the .msi installer for the JDK (screenshot below)
    • For OS X, scroll down to the MacOS section and download the .pkg installer for the JDK (screenshot below)
  4. Find the installer in your Downloads folder, or wherever you put it, and double-click, then follow the instructions. The default options should all be ok. (Make a note of the installation location, you may need it later.)
  5. After the installation completes, you may wonder if you can "run" it. You can't. It isn't an application, it's a programming language and an environment in which other Java-based applications can run. (Such as Eclipse.)

Installing Eclipse

Be sure you have installed the JDK first. Eclipse is a Java application and requires a Java installation in order to run.
  1. Go to https://www.eclipse.org/downloads/packages/installer
  2. Download the installer for your platform
  3. When you run the installer, select the package "Eclipse IDE for Java Developers" (screenshot below)

Trying it out

Find the Eclipse application (on a Mac, you'll find Eclipse.app in the Applications folder; on Windows, just use the search tool by the Windows button) and start it. When it starts, it will suggest a default location for your "workspace" where files and projects will be stored. Edit this as you see fit. Then try following the steps in the first Com S 227 lab.

If you have any problems, post a question on Piazza!

Alternative - use Remote Desktop

Most students in 227 prefer to have Java and Eclipse installed on their own computers and to keep their workspace on the local hard drive. The only challenge here is that we expect all your work to be securely backed up, so you'll need to be proactive in saving a copy of everything in network storage. We'll go over some options during Lab 1.

However, you can also get access to a Java/Eclipse installation using the CS department's terminal server via Remote Desktop. This will give you a virtual Windows desktop, similar to the physical labs we normally use in Pearson, and your files will be stored on the ISU servers.

Get on the VPN first

If you are outside the Iowa State campus network, you'll need to get on the VPN for remote access. Open up the IT services page: https://iastate.service-now.com/it/ and search for "VPN install" which should bring up the article entitled "Virtual Private Network (VPN): Install and Connect".

Connecting with Remote Desktop

(Mac users: you will need to install the Microsoft Remote Desktop client from the app store; it's free.)

The address is rdp.cs.iastate.edu. You'll need to log in using your ISU NetID and password. On Windows, when you enter your NetID, you'll need to prefix it with the IASTATE domain:

IASTATE\yourNetID

When you first create an Eclipse workspace, it will probably suggest a location on your U: drive. Normally, the drive letter U: is mapped to an ISU network server known more generally as myfiles. You can also access your myfiles storage directly as though it was a hard drive on your own computer. See this page https://www.cs.iastate.edu/how-access-your-files-through-samba-server for details.
